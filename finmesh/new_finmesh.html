FinMesh
FinMesh is a python-based package that brings together financial data from various sources in one place for ease of use and distribution. The four main sections of FinMesh are: (1) the IEX REST API, (2) data from the US treasury, (3) the data from the US Federal Reserve Economic Data, (4) and data from the SEC's EDGAR system.

The purpose of this package and its sub-packages was originally to practice interacting with API data. With third-party API packages there is always the risk of outages or bugs. In building the original IEX wrapper we sought to build something easy to understand and use, that can be updated quickly and accurately.

With the addition of the US Federal data the opportunity arose to create a package that could deliver all sorts of economic and security data from one place. In doing so we hope to create a low-barrier way for beginners to play with large and very useful data sets. In the future, this package will be updated with new financial and economic APIs.

If you know of a low or no cost API that could be incorporated please raise it as an issue and we will work to have it done ASAP.
Installation
FinMesh is available publicly from PYPI and can be installed and updated through a command line pip. Because the package is still growing and changing be sure to update often or check the PYPI page for possible updates.

There are several dependencies used in FinMesh, that soid, most of the package is built to be as stand-alone as possible. Most of the dependencies used are fairly standard, and you will be prompted to install when neccesary.

Some APIs require authentication through the use of tokens. These tokens should be set up as environment variables in the bash profile. I wrote a simple article outlining the setup on Mac and Linux which you can read through HERE.
IEX Functionality
IEX offers a vast array of financial and economic data. The REST API brings in data from many third party vendors as well as primary exchange data from IEX itself. Available data ranges from sentiment to Forex, and commodities to standardized financial statements. If you are looking to gather data for simple or algorithmic analysis, this is the best place to start.

An account is required to access the IEX API and most endpoints are only available to paid accounts. That said the lowest paid tier offers more than enough access for most individuals. In FinMesh, there are two main tokens that are used, a main key and a sandbox key.

Further, if you wish to make use of IEX's sandbox mode, you will need to create a boolean variable called SANDBOX and set it to True. A built-in parser will determine which token to use and make the adjustments for you.

The tokens should be stored in the following way as environment variables:

IEX_TOKEN=YOURTOKENHERE
and
IEX_SANDBOX_TOKEN=YOURSANDBOXTOKENHERE

FRED Functionality
The Federal Reserve of Economic Data (also know as FRED) maintains all the national economic data such as housing starts, non-farm payroll, and GDP. The available data is the broad and it is recommended you use the website to find the information you are looking for and then query the API through FinMesh. You are able to list available data through FinMesh.

EDGAR Functionality
The EDGAR system is the Security and Exchange Commission's database of all corporate filings including 10-Ks, change in ownership statements, and acquisition disclosures. The website is quite user friendly while programatic data colllection adds several layers of complexity. In it's current iteration, FinMesh is able to automatically find document types through plain text query and return any number of raw text, html, xbrli, and in some cases, xlsx filing information.

In the future the goal is to have complete, sanitized, and standardized information available for download in a csv or xlsx format for any document and every stock.
Compatibility with APIs
The name of the function shall be the name of the endpoint. The function shall accept all variables for a given endpoint using the same variable names per the documentation.

Differences between API documentation and this package should be considered errors. Please raise an issue if you notice discrepancies.

Specific documentation for every FinMesh function is not available but a list HERE can show you what options there are and the IEX docs will give you a full run down of the parameters. In general, to set a parameter you can set the parameter name equal to the desired output.
